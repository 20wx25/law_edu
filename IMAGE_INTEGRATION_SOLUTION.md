## 二、方案详细分析

### 方案1: Ink Tags + 自定义Web播放器 ⭐ 推荐

#### 原理说明

Ink原生支持Tags系统，用于在内容中嵌入元数据。Web播放器可以读取这些tags并据此显示图片。

**官方文档依据**:
- `WritingWithInk.md` 第98-106行: Tags基础语法
- `RunningYourInk.md` 第131-203行: Tags实现详解
- 官方示例: "用于图形冒险游戏中根据表情显示不同角色图"

#### 技术架构

```
Ink脚本 (tags) → JSON输出 → Web播放器读取tags → 显示对应图片
```

#### 实现步骤

**Step 1: 在Ink脚本中添加tags**
```ink
=== act1_after_school ===
# bg: 放学路_第一幕主背景.jpg
# char_left: 小雨平静版.png
# char_right: 孙伯伯和蔼版.png

放学铃声响起，同学们陆续离开教室。
```

**Step 2: Inky导出为Web**
- File → Export → Export story.js for web
- 自动生成包含tags的JSON数据

**Step 3: 修改Web播放器**
- 读取`story.currentTags`
- 解析tag中的图片文件名
- 动态更新HTML中的`<img>`元素

**Step 4: 部署测试**
- 将图片文件放到`graphics/`文件夹
- 在浏览器中打开HTML文件测试

#### 优点 ✅

1. **完全符合Ink官方设计理念**
   - Tags就是为了这种用途设计的
   - 不违反Ink语法和架构

2. **Ink脚本保持纯净**
   - 不需要外部函数绑定
   - 脚本仍可在Inky编辑器中正常运行

3. **灵活性极高**
   - 可以添加任意数量的tags
   - 支持背景、角色、特效、情绪等多维度控制
   - 动态tags（包含变量）也支持

4. **易于维护**
   - Tags位于内容旁边，一目了然
   - 修改图片只需改tag，不需要改代码

5. **可扩展性强**
   - 未来可以添加音效、动画等tags
   - 可以实现复杂的视觉效果

#### 缺点 ⚠️

1. **需要自定义Web播放器**
   - 默认播放器不处理图片tags
   - 需要编写JavaScript代码

2. **对HTML/CSS/JS有要求**
   - 需要一定前端开发知识
   - 但代码量不大（约100-200行JS）

3. **不能在Inky编辑器中预览图片**
   - 只能在Web播放器中看到效果
   - 测试流程稍长

#### 工作量评估

| 任务 | 预计时间 | 难度 |
|------|---------|------|
| 在v1.3脚本中添加tags | 2-3小时 | 低 |
| 修改Web播放器代码 | 3-4小时 | 中 |
| 测试和调试 | 2小时 | 低 |
| **总计** | **7-9小时** | **中** |

#### 示例代码

**Ink脚本部分**:
```ink
=== act1_after_school ===
# bg: 放学路_第一幕主背景.jpg
# char_left: 小雨平静版.png
# char_right: 孙伯伯和蔼版.png
# mood: calm

放学铃声响起，同学们陆续离开教室。
```

**JavaScript播放器代码**:
```javascript
function updateImages() {
    var tags = story.currentTags;

    tags.forEach(function(tag) {
        if (tag.startsWith('bg:')) {
            var bgFile = tag.substring(4).trim();
            document.getElementById('background').src = 'graphics/background/' + bgFile;
        }
        else if (tag.startsWith('char_left:')) {
            var charFile = tag.substring(11).trim();
            document.getElementById('char-left').src = 'graphics/characters/' + charFile;
        }
        // ... 其他tag处理
    });
}
```

### 方案2: Inky Web Export + 手动修改CSS/JS

#### 原理说明

使用Inky的"Export for web"功能，然后手动修改生成的HTML/CSS文件添加图片。

#### 实现步骤

1. 在Ink脚本中添加tags（同方案1）
2. Inky导出为Web
3. 手动编辑生成的HTML，插入`<img>`标签
4. 编写CSS控制图片位置和样式

#### 优点 ✅

1. 简单快速
2. 不需要复杂编程
3. 可以立即看到效果

#### 缺点 ⚠️

1. **每次修改脚本都要重新手动调整HTML**
2. **不够灵活，难以实现动态切换**
3. **维护成本高**

#### 结论

❌ **不推荐** - 适合一次性demo，不适合需要迭代的项目

### 方案3: 迁移到Twine或Ren'Py

#### 原理说明

放弃Ink，使用专门支持图片的互动小说引擎。

**Twine**:
- 开源网页互动小说引擎
- 原生支持图片、CSS样式
- 使用SugarCube或Harlowe格式

**Ren'Py**:
- 专业视觉小说引擎
- 原生支持图片、音效、动画
- Python脚本语言

#### 优点 ✅

1. 原生图片支持，无需额外开发
2. 功能更强大（存档、回放、音效等）
3. 社区资源丰富

#### 缺点 ⚠️

1. **需要完全重写924行脚本**
2. **学习新引擎和语法**
3. **浪费已完成的Ink工作**
4. **开发周期长（至少2-3周）**

#### 工作量评估

| 任务 | 预计时间 |
|------|---------|
| 学习新引擎 | 5-10小时 |
| 重写脚本 | 15-20小时 |
| 添加图片和测试 | 5小时 |
| **总计** | **25-35小时** |

#### 结论

❌ **不推荐** - 除非项目需要大量高级功能，否则不值得重新开始

### 方案4: External Functions (调用外部函数)

#### 原理说明

在Ink中声明外部函数，由JavaScript绑定实现图片显示。

**示例**:
```ink
EXTERNAL showImage(imageName)

=== act1_after_school ===
~ showImage("放学路_第一幕主背景.jpg")
放学铃声响起......
```

```javascript
story.BindExternalFunction("showImage", function(imageName) {
    document.getElementById('bg').src = 'graphics/' + imageName;
});
```

#### 优点 ✅

1. Ink官方支持的功能
2. 可以实现复杂逻辑

#### 缺点 ⚠️

1. **需要在每个需要图片的地方调用函数**
2. **脚本变得冗长，可读性下降**
3. **在Inky编辑器中运行会报错（除非定义fallback）**
4. **调试困难**

#### 对比Tags方案

| 特性 | Tags方案 | External Functions |
|------|---------|-------------------|
| 代码简洁性 | ✅ 简洁 | ❌ 冗长 |
| Inky兼容性 | ✅ 完全兼容 | ⚠️ 需要fallback |
| 可读性 | ✅ 高 | ❌ 低 |
| 开发效率 | ✅ 高 | ❌ 低 |

#### 结论

❌ **不推荐** - Tags方案在各方面都优于External Functions

## 三、最终决策

### 选择方案1: Ink Tags + 自定义Web播放器 ⭐⭐⭐⭐⭐

#### 决策理由

1. **完全符合Ink官方设计理念**
   - Tags系统就是为了图形冒险游戏设计的
   - 官方文档明确推荐这种用法

2. **Ink脚本保持纯净和可维护**
   - 不需要外部函数调用
   - 在Inky编辑器中仍可正常测试剧情逻辑

3. **开发成本合理**
   - 7-9小时的工作量可接受
   - 一次开发，长期受益

4. **扩展性强**
   - 未来可以轻松添加音效、动画等
   - Tags系统非常灵活

5. **符合Web发布需求**
   - 最终产品就是Web游戏
   - 自定义播放器可以实现任意视觉效果

## 四、技术规范预览

### Tags命名规范（详见下一文档）

```ink
# bg: 文件名.jpg          # 背景图片
# char_left: 文件名.png   # 左侧角色
# char_right: 文件名.png  # 右侧角色
# char_center: 文件名.png # 中央角色单人
# mood: 情绪标签          # calm/safe/tense/danger/sad/happy
# effect: 特效名称        # shake/darken/brighten/fade
# bg_filter: 滤镜        # darken/grayscale/blur
```

### 文件路径约定

