一、需求背景

1.1 现状问题与社会痛点

🚨 农村性侵案件高发且高度隐蔽

  • 案件数据触目惊心：2024年，侵害未成年人的高发罪名集中在强奸、猥亵儿童、强制猥亵等。公开及经核实的性侵儿童案件总量较上年上升12.7%
  • 城乡差异显著：农村地区显性案件占比达58%，但根据基层反馈，因"家丑不外扬"、"证据意识薄弱"而未报案的隐案比例可能高达70%
  • 真实案例：一位村医曾透露："村里有几个留守儿童出事，家长知道后，连夜带孩子外出打工，连派出所的门都没敢进。"

📚 现有教育资源严重错位

  • 主题错位：现有成熟的普法课程包主要集中在同学关系、校园霸凌等基础问题，与农村实际高发的性侵犯罪问题不符
  • 城市视角：现有教育资源多为城市视角设计，与农村实际情况（留守儿童多、熟人作案高发、救助资源匮乏）脱节
  • 内容过于简单直接：现有情景模拟往往过于简化，无法反映真实情况的复杂性（如权威人物施暴、受害者的心理困境等）

👩‍🏫 一线教师面临多重困境

困境类型 具体表现
内容敏感 性教育话题敏感，缺乏部门明确指导，教师不敢深入讲解
资源不足 没有配套的专业资料包，只能从网络或AI搜索，质量参差不齐
能力受限 教师本身未接受过专业培训，不知道如何系统教授防性侵知识
时间压力 课业压力大，没有时间深入备课，只能"点到为止"
侥幸心理 认为"不会发生在我们学校"，不重视预防教育

🏠 家庭层面的教育缺失

  • 话题禁忌：中国式家庭普遍忌讳谈论性相关话题，孩子在家中无法获得正确引导
  • 认知空白：农村家长自身对防性侵知识了解有限，无法教育孩子
  • 留守现象：大量留守儿童由祖辈抚养，代际沟通困难更加剧了教育缺失

1.2 为什么现在做？

⏰ 小学高年级到初中是关键窗口期

  • 认知基础已形成：这个年龄段的孩子已经有基础的认知能力，可以理解复杂情景
  • 风险行为尚在萌芽前：处于青春期前期或早期，正是建立安全意识的黄金时期
  • 已是高危受害群体：10-15岁是性侵案件的高发年龄段，预防教育刻不容缓
  • 干预效果最佳：在伤害发生前建立认知和应对能力，远比事后补救有效

💡 社会需求迫切

基层教育工作者强烈呼吁需要一套：

  • ✅ 符合农村实际情况的教育内容
  • ✅ 降低教师备课负担的标准化工具
  • ✅ 能够真正引发学生警觉和行动改变的教学方式
  • ✅ 提供持续学习和探索能力的配套资源

二、需求目标

目标类型 描述 衡量指标 目标值
核心教育目标 让学生能够准确识别哪些行为属于性侵（包括陌生人、熟人、权威人物等不同情景） 复盘阶段知识测试准确率 ≥80%
行为改变目标 学生遇到疑似情况时，知道第一时间应该做什么（告诉谁、怎么说、如何保护证据） 课后问卷"如果遇到XX情况你会怎么做"正确率 ≥70%
求助能力目标 如果已经发生侵害，学生知道如何寻求帮助（报警电话、救助机构、信任的成年人） 能准确说出至少2个求助渠道 ≥90%学生
用户体验目标 情景游戏要有足够的代入感，让学生能够真正"代入"而非敷衍完成任务 课后反馈"觉得情景贴近现实"的比例 ≥75%
传播效果目标 通过社交媒体和开源方式扩大影响力，触达更多教育工作者和关注者 社交媒体转发量、讨论度 （根据初期数据设定基线）
入校验证目标 通过大学资源与农村学校建立合作，完成真实课堂试点 成功入校试点数量 至少3所学校
教师减负目标 降低教师的备课负担，提供标准化、可直接使用的教学工具 教师反馈"比自己备课省时省力"的比例 ≥80%

⚠️ 关于长期效果的说明

关于"实际案件上报率提升"这一指标，我们认为：

  • 学生所处的生活环境差异很大，若本身就在比较安全的环境，没有案件上报的上升也是正常的
  • 案件上报率受多种因素影响（家长态度、社区文化、救助资源可及性等），难以归因于单一教育干预
  • 理想的验证方式是进行RCT对照研究（相似地理区域和人口经济环境的学校，一组使用本产品，一组不使用，对比效果），但实施难度较大
  • 因此，v1.0阶段不将此作为核心衡量指标，但在未来版本中可以考虑

三、用户与使用场景

3.1 典型用户画像

👧 主要用户：农村青少年学生

属性 特征描述
年龄范围 10-15岁（小学高年级到初中阶段）
地理分布 农村地区，部分为留守儿童
认知现状 对防性侵知识几乎零认知，家庭不谈论性话题，学校教育也"点到为止"
使用动机 完成学校课程要求（非主动寻求），但对"游戏"形式可能有好奇心
技术能力 基本的网页操作能力，熟悉智能手机
心理特点 对性话题可能感到尴尬，需要降低心理负担；对权威人物（老师、长辈）较为信任和服从

👨‍🏫 关键角色：一线教师

属性 特征描述
角色定位 课堂组织者、监督者、讨论引导者
核心诉求 有现成的、专业的、易用的教学工具，降低备课负担
能力现状 未接受过专业防性侵教育培训，缺乏系统知识和教学方法
使用场景 在课堂上带领全班学生使用产品，需要能实时看到学生的选择和AI对话内容
警觉需求 需要能够识别高风险学生（通过选择、提问等发现可能的受害者或潜在危险）

🌐 次要用户：网络测试用户

  • 用户群体：社交媒体上的成年网民、教育行业从业者、关注社会议题的公众
  • 使用动机：对产品形式感到好奇，想了解"情景游戏+AI普法"如何实现
  • 价值：提供产品功能性反馈，帮助团队了解产品与预期的差距，为入校前的优化提供依据

3.2 核心用户旅程图 (User Journey Map)

阶段 用户触点 用户行为 痛点 / 情绪 产品机会点
1. 课前准备 老师宣布要上一节"特殊的课" 学生听到要学"防性侵"，可能好奇、紧张或不当回事 😳 不知道要学什么，可能觉得尴尬或害羞 产品设计要降低心理负担，用"游戏"包装降低抗拒感；老师引导语要轻松但严肃
2. 进入情景 网页加载，展示情景背景画面和文字描述 阅读情景设定（时间、地点、人物关系），开始代入角色 🤔 如果太直白会不适，太委婉又无法理解真实风险 用"氛围和条件"暗示危险（如"空无一人的教室"、"他关上了门"），而非直接描述器官；通过AIGC生成的画面增强代入感
3. 做出选择 面对分支选项（2-4个选项） 根据直觉或已有认知做选择，可能和同学小声讨论 😰 不确定哪个选项是对的，担心选错被嘲笑；可能选择"听话""不惹事"等错误应对 不要在选择时即时判断对错（避免压力和羞耻感），让学生先完成全流程；设置"没有绝对错误"的选项，只有"更好"和"不够好"
4. 情景结束 看到选择导致的结果（如"你选择了保持沉默，施暴者变本加厉"） 可能惊讶、后悔或庆幸，开始反思自己的选择 😱 意识到"原来我的选择会导致这么严重的后果"或"原来应该这样做" 结果要有冲击力但不能太恐怖（避免二次创伤），用客观描述而非渲染恐惧；提供"正确做法会怎样"的对比
5. 复盘环节 看到正确做法解析、法律条文科普、求助渠道 理解为什么某些选项是对的/不够好，学习法律知识 🧐 希望记住关键信息但可能记不住；信息量大容易走神 用图文结合、重点标注、简洁话术（符合初中生理解水平）；提供知识测试环节强化记忆
6. AI助手探索 可以在文本框输入问题，AI实时回答 追问自己关心的问题（如"如果是我认识的人怎么办？"） 🤫 有些问题不敢当众问老师（怕暴露自己的经历或被认为"想太多"） AI提供安全的半匿名提问空间（老师可监督但不会立即打断），用温和的话术回应；对危机信号敏感并触发干预流程
7. 课后讨论 老师组织全班讨论，分享感受和疑问 部分学生发言，分享印象最深的情节或新学到的知识 😶 部分学生可能有创伤或正在遭受侵害，不愿发言但表情异常 老师需要敏感识别（产品可通过"选择模式"和"AI对话内容"为老师提供线索）；提供课后一对一交流的渠道

💡 关键洞察

  • 代入感是核心：只有真正"感同身受"，学生才会记住应对方法，否则就是"过眼云烟"
  • 不能过度恐吓：我们的目标是让学生警觉和应对，而非恐惧和创伤
  • 老师是关键节点：产品不能替代老师的观察和引导，要赋能老师而非取代老师
  • 隐私与监督的平衡：学生需要安全空间提问，但老师也需要知道谁可能处于危险中

四、需求功能清单

4.1 模块一：情景模拟核心

功能点 描述 优先级
多分支剧情系统 基于真实案例改编的剧本，包含多个关键决策点。施暴者角色多样化（陌生人在公共场所、熟人在家中、权威人物利用职权等），避免职业歧视 P0
选择机制 在关键节点提供2-4个选项，不同选项导向不同的情节发展和结果。选项设计要符合学生的认知水平和实际可能的反应 P0
法律条款嵌入 在选项阶段或情景结束后的复盘阶段，插入相关法律科普（如《未成年人保护法》《刑法》相关条款），用通俗语言解释 P0
视觉呈现（AIGC图像） 使用AIGC生成角色和场景图像，配合文字叙述增强代入感。画风要适合青少年（不过于幼稚也不过于成人化） P0
氛围营造 通过环境描述（"空无一人的教室"、"他关上了门"、"夜幕降临"）和条件暗示（"他说这是秘密，不能告诉别人"）来传达危险信号，而非直接描述侵害行为 P0
背景音效/音乐（可选） 轻微的背景音乐或环境音效（如雨声、脚步声）营造氛围，增强沉浸感 P1

4.2 模块二：复盘与教学系统

功能点 描述 优先级
选择回顾 展示学生刚才的选择和导致的结果，与"如果选择其他选项会怎样"进行对比 P0
正确做法解析 说明在类似情况下应该如何应对（第一时间、中期、长期应对），强调"你没有错，错的是施暴者" P0
法律知识科普 解释相关法律条文（刑法、未成年人保护法），说明施暴者会受到什么惩罚，受害者有什么权利 P0
求助渠道指引 列出报警电话（110）、当地救助机构、心理热线、信任的成年人（家长、老师）等求助渠道，并说明"找谁、怎么说" P0
知识测试环节 2-3道选择题检验学习效果（如"如果遇到XX情况，最应该先做什么？"），即时反馈对错并解释 P1

4.3 模块三：AI助手系统

功能点 描述 优先级
RAG知识库 包含法律条文、心理疏导话术、求助机构联系方式、地域化信息（不同省份的报警流程、当地救助机构） P0
自由问答 学生可以在文本框输入任何问题，AI基于知识库实时回答。回答要符合青少年理解水平，避免法律术语堆砌 P0
超范围引导 当学生的问题超出知识库范围（如"我该不该原谅侵犯我的人"），AI应引导学生找老师或专业心理咨询师，不要给出主观建议 P0
危机干预机制 识别到"我正在遭受性侵"、"这件事发生在我身上"等危机信号时，立即提供求助热线、安抚学生、记录对话并通知相关机构（老师、学校） P0（高风险）
对话记录（老师可查看） 老师可以看到学生与AI的对话内容（用于监督和识别高风险学生），但不会在课堂上公开，保护学生隐私 P1

4.4 模块四：教师管理后台（未来扩展）

功能点 描述 优先级
课堂组织功能 生成课堂码或链接，学生扫码/点击进入，老师可以统一控制进度（如"全班一起进入情景""现在开始复盘"） P2
实时监控面板 老师可以看到每个学生当前的选择、进度、AI对话内容（用于识别异常或高风险学生） P2
数据统计报告 课后生成报告：全班知识测试准确率、常见错误选项、高频提问、学生反馈等，帮助老师了解教学效果 P2
高风险提示 如果AI识别到危机信号（学生透露正在遭受侵害），立即通知老师（弹窗或短信），老师可以在课后单独关注该学生 P0（功能重要，但技术实现可能在P2）

七、异常与边界处理

异常场景 系统如何处理 / 提示文案
网络断开 提示："当前网络不佳，请稍后重试。如果在课堂上遇到此问题，请告诉老师。"（情景进度已自动保存在本地，刷新后可继续）
学生在AI对话中透露危机信号 AI立即回应："我注意到你可能正在经历困难。请立即告诉你信任的成年人（老师、家长），或拨打110报警 / [当地救助机构电话]。你没有错，我们会保护你。"同时记录对话并通知老师
学生提问超出知识库范围 AI回应："这是一个很重要的问题，但超出了我目前的能力范围。建议你课后和老师或专业心理咨询师深入讨论。"
学生在情景中反复选择"错误"选项 不立即打断（避免羞耻感），让学生完成全流程后在复盘环节重点解释；如果多次选择明显不合理的选项（如"主动接近施暴者"），老师监控面板会标记该学生，建议课后单独交流
家长投诉内容"过于露骨" 入校前必须通过专家审核，确保内容符合年龄段；向家长提供"致家长的一封信"，解释防性侵教育的重要性和内容设计原则；强调"氛围暗示而非直接描述"的设计理念
学校认为话题"太敏感" 准备数据和案例说服（农村性侵高发、隐案率70%、窗口期的重要性）；提供试用版让教职工先体验；可根据学校反馈调整部分表述，但不降低教育效果
学生课后反馈"没学到东西" 在复盘环节增加知识点数量和深度；增加知识测试环节强化记忆；调整情景设计，确保"冲击力"足够
教师反馈"操作太复杂" 简化教师端操作流程（如一键生成课堂链接、自动进度同步）；提供操作视频教程；提供技术支持联系方式

🚨 关于危机干预的特别说明

当前团队未配备心理咨询或社工专业背景成员，因此AI的危机干预话术必须：

  • ✅ 使用经过验证的、来自专业机构的标准话术（如联合国儿童基金会、中国少年儿童基金会等）
  • ✅ 明确告知求助渠道（110、当地未成年人保护热线），不替代专业干预
  • ✅ 在入校前，必须邀请专业人士（心理咨询师、社工）审核这部分内容
  • ❌ 避免给出主观建议（如"你应该原谅他"、"这不是什么大事"等）
  • ❌ 避免过度承诺（如"我会帮你解决"），明确AI的能力边界

八、数据追踪与埋点

8.1 核心数据埋点

埋点位置 触发时机 追踪指标 目的
情景进入 学生点击"开始情景" 进入率、退出率、平均停留时长 评估情景的吸引力（如果大量学生刚进入就退出，说明开头不够吸引人）
选择节点 学生在每个分支点做出选择 各选项选择比例、犹豫时长（从展示到点击的时间） 分析学生的认知误区（如80%学生选择"保持沉默"，说明需要重点强化"勇敢求助"）
复盘环节 学生查看复盘内容 阅读完成率、知识测试准确率 评估教学效果（如果知识测试准确率低，说明复盘内容需要优化）
AI提问 学生向AI助手提问 提问数量、高频问题、超范围问题比例 了解学生的关注点和困惑点，优化知识库内容
危机信号触发 AI识别到危机关键词 触发次数、老师响应时间 评估危机干预机制的有效性；如果频繁误触发，需要调整识别规则
课后反馈 课程结束后弹出问卷 满意度评分、"觉得情景贴近现实"比例、建议文本 收集主观反馈，指导产品迭代

8.2 隐私与数据安全

  • 学生个人信息：不收集学生的真实姓名、身份证号等敏感信息；如果需要区分学生（如生成报告），使用匿名ID
  • AI对话记录：老师可查看但不可导出或分享；如果学生透露危机信号，该对话应被特别标记并按规定上报
  • 数据存储：学生选择和对话数据应加密存储，课程结束后根据学校要求决定保留时长或删除
  • 合规要求：入校前需了解《个人信息保护法》《未成年人保护法》对数据收集的要求，必要时签署数据处理协议

💡 关于开源策略

我们计划将产品以开源方式发布（如GitHub），原因：

  • ✅ 降低推广成本：让任何关注普法教育的组织或个人都能使用和改进
  • ✅ 吸引贡献者：鼓励法律、教育、心理等领域的专业人士贡献内容和建议
  • ✅ 建立信任：开源透明，让学校和家长放心产品内容
  • ✅ 社区验证：通过"自来水用户"的反馈快速迭代

开源协议：建议使用MIT或GPL协议，允许商业使用但要求衍生作品也开源。
